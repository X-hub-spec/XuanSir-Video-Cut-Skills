# 情绪价值功能 TODO

需求来源：`019e4f66-9130-79b3-82ee-247accd65aac`

目标：让口播粗剪审核页不只是工具，还能在剪辑过程中提供陪伴感、反馈感和隐藏奖励，让长时间删废话变得更有趣。

## 已确认 TODO

### 1. 每剪掉 5% 触发一次《道德经》鼓励

- 按「已确认删除时长 / 原视频总时长」计算删减进度。
- 每跨过一个 5% 节点触发一次鼓励，例如 5%、10%、15%、20%。
- 每个节点只触发一次，避免用户来回取消/恢复选择时重复刷屏。
- 鼓励角色暂定为「老子」出现。
- 老子的鼓励文案使用《道德经》原著句子，不写现代鸡血文。
- 文案要短、意味深长、能和剪辑动作产生隐喻关系，适合出现在工作流中，不要打断操作。
- 老子的头像需要单独生成，不使用老板头像。
- 老子头像方向：道家、古人、智慧、克制、像素/赛博风可以和老板头像同台，但气质更沉静。

示例文案：

- `天下难事，必作于易；天下大事，必作于细。`
- `合抱之木，生于毫末；九层之台，起于累土。`
- `为学日益，为道日损。`
- `少则得，多则惑。`
- `大音希声，大象无形。`
- `飘风不终朝，骤雨不终日。`

适配剪辑场景的触发解释：

- 剪掉 5%：`天下难事，必作于易；天下大事，必作于细。`
- 剪掉 10%：`为学日益，为道日损。`
- 剪掉 15%：`少则得，多则惑。`
- 剪掉 20%：`大音希声，大象无形。`
- 清掉大段废稿：`合抱之木，生于毫末；九层之台，起于累土。`
- 清理连续停顿：`飘风不终朝，骤雨不终日。`

### 2. 4 分钟无动作触发老板意味深长地鞭策

- 如果连续 4 分钟没有有效操作，老板出现并鞭策。
- 有效操作包括：选择/取消删除、拖选、播放/暂停、跳转、快捷键操作、导出/渲染前的审核操作。
- 老板出现后，不自动立刻消失。
- 当用户持续恢复工作 30 秒后，老板消失。
- 「持续工作 30 秒」的判定：过去 30 秒内累计至少 5 秒存在操作活动。
- 用户描述里的「老板小时」按「老板消失」理解。
- 老板头像使用用户提供图片：`/Users/xuansir/Library/Mobile Documents/com~apple~CloudDocs/个人相关/赛博轩轩头像/第二版.jpeg`。
- 老板的语气不要直白骂人，也不要普通催促，要有意味深长、似笑非笑、压迫感轻但后劲足的感觉。
- 老板文案应该像“话不重，但用户知道自己该动了”。

示例文案：

- `老板：你停下来的这四分钟，视频倒是很完整。`
- `老板：有些沉默是留白，有些沉默只是还没开始。`
- `老板：素材没有意见，它只是等你做决定。`
- `老板：你看了它很久，它也看了你很久。`
- `老板：删不删都可以，时间会替你保留原样。`
- `老板：真正难剪的，往往不是视频。`

## MVP 优先级

1. 先做「状态统计与触发器」
- 记录删减百分比。
- 记录已触发的 5% 节点。
- 记录最近操作时间与操作活跃窗口。

2. 再做「轻量角色出现」
- 不做复杂动画，先做一个右下角小浮层。
- 老子浮层展示头像 + 《道德经》原句，自动 2-4 秒消失。
- 老板鞭策浮层持续显示，直到恢复工作条件满足。
- 老板浮层展示用户提供的赛博轩轩头像。

3. 最后做「隐藏奖励」
- 渲染完成后展示本片抢救报告。
- 里程碑、连续处理、长段删减都可以解锁不同隐藏奖励。
- 奖励只做轻量反馈，不阻塞剪辑流程。

## 方案库

### A. 剪辑成就系统

把剪辑过程拆成隐藏成就，用户达成后才出现。

- `废话清洁工`：累计剪掉 5%。
- `节奏医生`：累计剪掉 15%。
- `静音猎手`：删除 10 段以上静音。
- `重说终结者`：连续处理 20 个 AI 建议。
- `开头救星`：剪掉开头 10 秒内的大段废稿。
- `一刀入魂`：一次 Shift 拖选超过 8 秒内容。
- `完片前夜`：选中删除时长超过 AI 建议的 80%。

奖励形式：

- 小浮层徽章。
- 审核页标题短暂变色。
- 底部状态栏出现一句彩蛋文案。
- 渲染完成页展示本次成就。

### B. 角色陪伴系统

页面里有两个角色：

- `老子`：负责用《道德经》原著给出克制、玄妙、意味深长的鼓励。
- `老板`：负责空闲提醒、轻度压迫、把用户拉回工作。

触发规则：

- 老子：进度里程碑、连续完成、渲染成功。
- 老板：空闲过久、打开页面后长时间不开始、选了很多但迟迟不渲染。

注意：

- 角色不要频繁说话，避免烦。
- 文案要短，最好 12-24 字。
- 允许用户一键静音/隐藏角色。
- 老子和老板的头像风格要能同台出现，但性格区分明确：老子沉静，老板锐利。

### C. 隐藏奖励

适合放在用户达成阶段目标后出现。

- 剪掉 10% 后解锁「今日手感」评价。
- 剪掉 20% 后解锁「节奏评分」。
- 渲染成功后出现「本片抢救报告」。
- 连续工作 25 分钟后出现「专注勋章」。
- 没有触发老板提醒就完成渲染，奖励「自驱剪辑师」。

渲染完成奖励页示例：

- 原片时长
- 成片时长
- 剪掉时长
- 节省百分比
- 处理静音数量
- 处理重复/废话数量
- 今日称号

### D. 温和防打扰机制

情绪价值不能影响剪辑效率。

- 浮层不遮挡文稿和视频关键区域。
- 老板提醒出现频率有限制，比如 15 分钟内最多 2 次。
- 鼓励浮层可点击关闭。
- 所有奖励都不阻止用户继续操作。

## 实现草案

### 状态变量

```js
const moraleState = {
  triggeredMilestones: new Set(),
  lastActionAt: Date.now(),
  actionEvents: [],
  bossVisible: false
};
```

### 操作记录

每次用户做有效操作时调用：

```js
function recordUserAction() {
  const now = Date.now();
  moraleState.lastActionAt = now;
  moraleState.actionEvents.push(now);
  moraleState.actionEvents = moraleState.actionEvents.filter(t => now - t <= 30000);
}
```

### 5% 里程碑

在 `updateStats()` 或选择变化后检查：

```js
function checkDeleteMilestone() {
  const duration = player.duration || 0;
  if (!duration) return;
  const percent = getSelectionDuration(selected) / duration * 100;
  const milestone = Math.floor(percent / 5) * 5;
  if (milestone >= 5 && !moraleState.triggeredMilestones.has(milestone)) {
    moraleState.triggeredMilestones.add(milestone);
    showLaoziQuote(milestone);
  }
}
```

### 老板提醒

每 5 秒检查一次：

```js
setInterval(() => {
  const now = Date.now();
  if (!moraleState.bossVisible && now - moraleState.lastActionAt >= 4 * 60 * 1000) {
    showBoss();
  }
  if (moraleState.bossVisible) {
    const recentActions = moraleState.actionEvents.filter(t => now - t <= 30000);
    const activeEnough = recentActions.length >= 5;
    if (activeEnough) hideBoss();
  }
}, 5000);
```

备注：如果要严格实现「过去 30 秒有 5 秒的操作」，需要把操作建模为区间，而不是次数。MVP 可以先用高频操作次数近似，后续再精细化。
