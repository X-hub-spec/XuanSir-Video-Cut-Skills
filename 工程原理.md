# 工程原理

XuanSir Video Cut Skills 的核心思路是：**用文字稿驱动视频粗剪**。

传统剪辑要在时间线上找口误、拖切点、对齐字幕。这个工程把视频先拆成带时间戳的文字 token，再让用户在文稿里确认删除内容，最后把这些文字选择反推成视频剪辑区间。

## 1. 从视频到字级 token

系统先用 FFmpeg 提取音频，再通过本地 Whisper MLX 或火山引擎 ASR 做语音识别。

识别结果会被整理成 `subtitles_words.json`：

- 每个字/词都有 `start` 和 `end` 时间。
- 静音也会被写成独立 gap token。
- segment 原文会保留，用于生成标点、段落和字幕行。

这一步让后面的所有操作都有统一坐标：**原视频时间线上的 token index**。

## 2. AI 只做预选，用户做最终确认

AI 会根据用户习惯和规则分析口播内容，生成 `auto_selected.json`。

它主要寻找：

- 长停顿
- 重复句
- 句内重复
- 说错后重说
- 残句
- 口头填充词

这些结果在审核页里只是“AI 建议”，不会直接等于最终剪辑。用户确认后，token 才会进入红色删除状态。

## 3. 审核页是一个 token 编辑器

网页右侧看起来像文章或字幕，但底层仍然是 token DOM。

每个可剪辑元素都保留 `data-index`，所以这些操作都能映射回原始时间：

- 单击跳转视频
- 双击删除/取消
- Shift 拖动画线批量选择
- 修改文字
- 字幕分段/合并
- 撤销/重做

文章视图适合通读粗剪，字幕视图适合检查每条字幕的长度、换行和节奏。两个视图共用同一份 token 状态。

## 4. 删除选择如何变成剪辑

用户确认删除后，系统会把连续红色 token 合并成删除区间。

然后反推出保留区间：

```text
原视频时间线 - 删除区间 = 保留区间
```

渲染时，FFmpeg 只拼接保留区间，所以最终视频就是“留下来的部分”。

播放预览时也会使用同样的删除区间：播放头进入红色区间后自动跳到区间结束，让用户能提前感受剪后节奏。

## 5. FCPXML 和 SRT 为什么能对齐

导出 FCPXML 时，系统把保留区间写成 Final Cut Pro 可导入的时间线。

导出 SRT 时，系统不会使用原视频时间码，而是把字幕映射到剪后的新时间线：

```text
原始 token 时间 -> 所在保留片段 -> 剪后累计时间
```

因此 SRT 会从 `00:00` 开始连续排列，和 FCPXML 的剪后时间线一致。红色删除内容不会出现在 SRT 里，系统自动补的阅读标点也不会进入导出的 SRT。

## 6. 本地项目状态

审核页会把项目状态保存到本地：

- 已确认删除的 token
- 文稿修改
- 字幕分段和合并
- 当前视图
- 播放位置
- BGM 选择

重新打开审核页时，系统读取本地状态并恢复。这样粗剪可以像一个真实工程一样分多次完成。

## 7. BGM 为什么不会进入成片

BGM 只在审核页作为预览参考音轨。

它跟随视频播放和暂停，但不会写入：

- FFmpeg 渲染输入
- FCPXML
- SRT

这样用户可以用音乐判断节奏，同时最终交付仍保持原视频和原音轨。

## 8. 多机位模式

多机位模式仍然沿用“文稿驱动粗剪”，但把底层坐标升级为全局稿件时间轴。

输入是 `multicam_sources.json`：每个视频或音频素材有 `id`、`path`、`kind`、`offset`，说话人通过音频轨的 `speakerId` 绑定。准备脚本会为素材建立 `media/` 符号链接，按带说话人的音轨生成字级 token，并合并成 `multicam_project.json`。

多机位 token 仍然有 `start` 和 `end`，但它们表示全局时间。每个说话 token 还带有 `speakerId` 和 `sourceId`，分别用于文稿显示和导出映射。

审核页里的删除仍然是 token 选择。段落移动则是把一个全局时间区间放到新的时间线顺序里。导出 FCPXML 时，系统会按段落顺序生成保留片段，用 `sourceLocalTime = globalTime - source.offset` 映射回每条素材，并把主机位写在 spine 上、其他视频/音频写成不同 lane 的多轨片段。

首版多机位模式不做自动声纹识别、不做自动波形同步，也不直接渲染 mp4。它的目标是快速得到可进 Final Cut Pro 精修的多轨工程。

## 总结

这个工程的关键不是“自动剪视频”，而是建立一个可靠的中间层：

```text
视频声音 -> 字级 token -> 文稿审核 -> 删除区间 -> 剪后时间线
```

只要 token 时间戳可靠，文稿上的每一次划线、修改和分段，都可以稳定地转换成视频剪辑结果。
