# Videocut Skill 便携包使用说明

这是一套本地口播粗剪 Skills：转录、AI 口误分析、网页审核、渲染、导出 FCPXML、BGM 试听都在里面。

## 1. 安装到另一台电脑

把压缩包解压后，得到 `videocut/` 文件夹。把它放到目标电脑的 Skills 目录：

```bash
mkdir -p ~/.claude/skills
cp -R videocut ~/.claude/skills/videocut
```

如果目标工具使用的是 Codex 自己的 skills 目录，也可以放到：

```bash
mkdir -p ~/.codex/skills
cp -R videocut ~/.codex/skills/videocut
```

建议两边都放一份，Claude Code 和 Codex 都能更容易识别。

## 2. 安装依赖

macOS 推荐：

```bash
brew install node ffmpeg
cd ~/.claude/skills/videocut
python3 -m venv .venv
.venv/bin/python -m pip install -U pip
.venv/bin/python -m pip install mlx-whisper
```

本地转录默认使用 `mlx-community/whisper-small-mlx`。第一次运行会下载模型，时间取决于网速。

如果不用本地 Whisper，也可以配置火山引擎 ASR：

```bash
cd ~/.claude/skills/videocut
cp .env.example .env
# 编辑 .env，填入 VOLCENGINE_API_KEY
```

## 3. 使用方式

在 Codex 或 Claude Code 里，把视频文件交给它，然后说：

```text
帮我剪这个口播视频
```

典型流程是：

1. 提取音频。
2. 本地 Whisper MLX 转录。
3. 生成字级时间戳。
4. AI 分析口误、重复、长停顿。
5. 生成审核网页。
6. 在网页里读稿、划线、确认删除。
7. 点击 `渲染` 输出保留后的成片。

审核页通常在：

```text
output/日期_视频名/剪口播/3_审核/review.html
```

服务器启动命令类似：

```bash
cd output/某个项目/剪口播/3_审核
node ~/.claude/skills/videocut/剪口播/scripts/review_server.js 8899 video.mp4
```

然后打开：

```text
http://localhost:8899/review.html
```

## 4. 审核页怎么用

- 单击文字：跳到对应视频时间。
- 双击文字或停顿：切换删除/保留。
- 按住 Shift 拖过文字：像划线一样批量删除或取消。
- 红色删除线：最终会被剪掉。
- 橙色虚线：AI 建议。
- `保存项目`：把当前红线、播放位置、BGM 选择保存到本地。
- 再次打开网页：自动恢复上次剪辑状态。
- `导出 FCPXML`：导出 Final Cut Pro 可导入的 XML。
- `渲染`：只渲染留下的部分。

## 5. BGM

BGM 放在：

```text
剪口播/BGM/
```

审核页会自动扫描这个目录。新增或改名 BGM 后，重新生成审核页即可出现在下拉菜单。

注意：BGM 只是审核时参考，不会进入最终渲染视频，也不会进入 FCPXML。

## 6. 这个包里有什么

```text
videocut/
├── README.md
├── .env.example
├── 使用说明.md
├── 安装/
├── 剪口播/
│   ├── BGM/
│   ├── scripts/
│   ├── 用户习惯/
│   └── SKILL.md
├── 字幕/
├── 高清化/
└── 自进化/
```

这个 zip 不包含：

- 当前电脑上的输出视频。
- Whisper 模型缓存。
- Python 虚拟环境。
- Git 仓库历史。
- 私密 API Key。

这些都需要目标电脑按需重新生成或配置。
